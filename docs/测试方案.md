# UniChat 测试方案

> 适用范围：v1（Bot-only + 最小转人工）。本方案是对现有测试体系的**补位**，不是重写。

## 1. 现状盘点

截至本文档落笔，项目已有：

| 层 | 工具 | 覆盖 | 位置 |
|---|---|---|---|
| 单元/集成 | pytest + pytest-asyncio + httpx ASGITransport | 243 个测试：slice3-6、webhook/reply 全链路、Telegram/WhatsApp/Test adapter、admin UI 的 HTML 字符串断言、WS notifier | `tests/`（扁平） |
| 静态检查 | ruff + mypy（strict） | src + tests | `pyproject.toml` |
| 部署冒烟 | curl `/health` | 生产健康端点 | `.github/workflows/deploy.yml` post-deploy |

**关键事实**：`tests/test_admin_ui.py` 已经用 httpx 跑通了登录、会话列表、消息时间线、人工回复、resolve 的 HTML 断言。这些是**集成层**测试，又快又稳，**不要用 Playwright 重写它们**。

**httpx 测不到、必须 E2E 补的**：
- htmx 局部 swap（点会话行 → `#message-panel` 替换；回复表单 submit → 新气泡出现）
- WebSocket 推送（`message.created` → `htmx.ajax` 重载 `#conv-list`，无整页刷新）
- 真实浏览器行为（cookie/redirect、JS console error、`ws.onclose → 5s reload` 的重连循环）
- 视觉/布局（DESIGN.md 已有设计系统，可做轻量视觉回归）

## 2. 对外部 GPT 方案的取舍

| GPT 建议 | 本项目处理 | 理由 |
|---|---|---|
| pytest-playwright（Python 版，不引 Node） | ✅ 采纳 | 栈全 Python，无意义引 Node |
| `data-testid` 定位 | ✅ 采纳，按清单增量加 | 现模板只有 `#conv-list`/`#message-panel`/`#conv-status` 三个 id，交互元素无 testid |
| `tests/unit/api/e2e` 三级目录重构 | ❌ 不采纳 | 已有 243 个扁平测试 + 部署钩子依赖扁平；新增 `tests/e2e/` 子目录即可 |
| `.env.test` + `DATABASE_URL` 切库 | ❌ 不采纳 | 现有用 `init_db("sqlite://")` autouse fixture；E2E 用临时文件库 + 测试 config 注入，不走环境变量 |
| 登录用 `username/password` | ❌ 不适用 | 实际是单 `token` 字段，照抄会全红 |
| 重写登录/列表/回复 E2E | ❌ 不重写 | `test_admin_ui.py` 已覆盖 HTML 断言；E2E 只补 htmx/WS/浏览器层 |
| Playwright MCP 作为核心组件 | ⚠️ 降级为 Phase 3 可选 | MCP 是开发期 AI 探索工具，不进测试架构 |
| 重配 `pyproject.toml` 的 ruff/mypy | ❌ 不动 | 已有 `strict=true` 等配置，GPT 版本会覆盖回退 |
| 部署后 smoke test | ✅ 已有 `curl /health`；Playwright smoke 列为可选 | v1 不必加 |

## 3. 测试分层总览

```
┌─────────────────────────────────────────────────────────┐
│ L1 单元/集成   pytest + httpx ASGITransport   <1s        │  已有，不动
│  - 服务层、adapter、bus、admin HTML 字符串断言           │
│  - 内存 SQLite，autouse fixture 隔离                     │
├─────────────────────────────────────────────────────────┤
│ L2 E2E 浏览器  pytest-playwright            ~5-15s       │  本方案新增
│  - htmx swap / WebSocket push / 真实浏览器行为           │
│  - 临时文件 SQLite + 测试 config + 线程内 uvicorn.Server │
│  - @pytest.mark.e2e 标记，默认跳过，不进部署钩子         │
├─────────────────────────────────────────────────────────┤
│ L3 生产冒烟    curl /health                 <1s          │  已有
│  - Playwright smoke against 生产 URL：可选，v1 不强求    │
└─────────────────────────────────────────────────────────┘
```

**原则**：L1 不动，L2 只补 L1 测不到的，L3 维持现状。**不追求"全部 E2E"**——SSR + htmx 项目里，httpx HTML 断言的性价比远高于浏览器 E2E。

## 4. E2E 补位范围（只列该补的）

| # | 流程 | 为什么 httpx 测不了 | 优先级 |
|---|---|---|---|
| E1 | 登录 → 真实 302 + cookie 落地 + URL 变 `/admin` | httpx 不跑 JS、不显示 URL 跳转链 | P0 |
| E2 | 错误 token → 错误提示渲染、URL 停在 `/admin/login` | httpx 已测 HTML，但浏览器层再确认一次无副作用 | P1 |
| E3 | 点会话行 → htmx GET `/admin/conversations/{id}/messages` → `#message-panel` swap 出气泡 | htmx 是客户端 JS，httpx 完全无法触发 | **P0** |
| E4 | 回复表单 submit → htmx POST → `#message-panel` swap 出新 user 气泡 | 同上，htmx 驱动 | **P0** |
| E5 | resolve → htmx POST → `#conv-status` badge 变 `resolved` + 按钮禁用 | 同上 | P0 |
| E6 | WS 推 `message.created` → `#conv-list` 自动刷新出新会话，无整页 reload | WebSocket + htmx.ajax，httpx 无法模拟 | **P0** |
| E7 | `ws.onclose` → 5s 后 `location.reload()` 不死循环 | 真实浏览器计时器 + 重连语义 | P1 |
| E8 | 整个会话期间 `page.console_errors == []` | 捕获 htmx/WS/模板 JS 错误，httpx 看不到 | P0 |
| E9 | 移动端视口（375px）下 `.admin-layout` 不溢出、`.conv-list` 320px 不挤压 | DESIGN.md 有设计系统，做轻量视觉断言 | P2 |
| E10 | 未登录直访 `/admin` → 302 回 login | httpx 已测，浏览器层仅作护栏 | P2 |

**不做的**：登录/列表/回复的纯 HTML 内容断言——`test_admin_ui.py` 已覆盖，E2E 只断言"swap 发生了、新元素可见、console 干净"，不断言具体文案。

## 5. 技术选型

- **`pytest-playwright`**（Python 版，同步 API 优先）：与现有 pytest 同进程，无需 Node。
- **同步 API**：项目测试以 `pytest-asyncio` 为主，但 Playwright 同步 API 在 E2E 里更直白；E2E 用独立 `conftest.py` 不与 async fixture 混用。
- **不引 Playwright MCP 进测试架构**：MCP 是给 AI 在开发期探索页面用的，跑测试时不需要。是否装 MCP 留给开发者个人决定（见 §11 Phase 3）。
- **不引 `pytest-fastapi` 之类额外依赖**：E2E server 用 `uvicorn.Server` 在线程内启动，复用 `src.app.create_app`。

### 6.2 隔离策略

E2E 需要一个**TCP 可达**的 server，内存 SQLite 跨进程不可达，因此用**临时文件 SQLite**：

```python
# tests/e2e/conftest.py（示意，落地时按此结构实现）
import pytest, threading, uvicorn
from pathlib import Path

@pytest.fixture(scope="session")
def e2e_server():
    db_path = Path(tmp_path_factory.mktemp("e2e")) / "test.db"
    init_db(f"sqlite:///{db_path}")
    create_all()

    config = AppConfig(
        inboxes=[InboxConfig(id="tg", channel_type="test", config={...})],
        server=ServerConfig(host="127.0.0.1", port=0, admin_token="e2e-token"),
        database_url=f"sqlite:///{db_path}",
    )
    app = create_app(config)
    server = uvicorn.Server(uvicorn.Config(app, host="127.0.0.1", port=0, log_level="warning"))
    # 用 server.config.bind 拿实际端口；或在启动后读 server.servers[0].sockets[0].getsockname()
    t = threading.Thread(target=server.run, daemon=True)
    t.start()
    yield base_url  # http://127.0.0.1:<actual_port>
    server.should_exit = True
```

要点：
- `port=0` 让 OS 分配空闲端口，避免并行/CI 端口冲突。
- `channel_type="test"`（项目已有 Test adapter，见 `tests/test_test_adapter.py`）——E2E 不依赖真实 Telegram API。
- `scope="session"`：server 整个 session 复用；**每个测试函数用 fixture 清表**（`truncate` 或 drop/create），不复用数据。
- 不走 `src/main.py:app` 模块级全局——那个读 `config.yaml` + 真实 env，不适合 E2E。

## 7. `data-testid` 改造清单

只给**交互元素**加，不给纯文本加。改动范围小，逐文件列：

### `login.html`
```html
<input data-testid="login-token" name="token" ...>
<button data-testid="login-submit" type="submit">登录</button>
<div data-testid="login-error" ...>{{ error }}</div>   <!-- 始终渲染，error 为空时隐藏 -->
```

### `admin.html`
```html
<div class="conv-list" id="conv-list" data-testid="conv-list">
  ...
  <div class="conv-row" data-testid="conv-row" data-conv-id="{{ conv.id }}" ...>
```
> `data-conv-id` 让测试能定位"第 N 个会话"并取 id 拼 htmx URL。

### `_messages.html`
```html
<span data-testid="conv-status" ...>{{ conversation.status }}</span>
<button data-testid="resolve-btn" ...>关闭对话</button>
<!-- 每条气泡 -->
<div data-testid="msg-bubble" data-sender="{{ msg.sender_type }}" ...>
<input data-testid="reply-input" name="content" ...>
<button data-testid="reply-submit" type="submit">发送</button>
<div data-testid="reply-error" ...>{{ reply_error }}</div>
```

### `base.html`
无需 testid（仅布局）。

**原则**：加 testid 不改现有 class/id，避免影响现有 CSS 与 httpx 测试里的 `b"..." in resp.content` 断言。

## 8. 必测流程落地示例

E3（点会话行 → htmx swap）示范写法：

```python
@pytest.mark.e2e
def test_click_conversation_loads_messages(page, e2e_server, seeded_conversation):
    page.goto(f"{e2e_server}/admin/login")
    page.get_by_test_id("login-token").fill("e2e-token")
    page.get_by_test_id("login-submit").click()
    page.wait_for_url("**/admin")

    page.get_by_test_id("conv-row").first.click()
    # htmx swap 完成的信号：气泡容器出现
    page.get_by_test_id("msg-bubble").first.wait_for(state="visible", timeout=3000)
    bubbles = page.get_by_test_id("msg-bubble").all()
    assert len(bubbles) >= 1
    assert page.get_by_test_id("reply-input").is_visible()
```

E6（WS 推送）示范：

```python
@pytest.mark.e2e
def test_ws_push_refreshes_conv_list(page, e2e_server, e2e_db_session):
    # 登录 + 进 admin，等 WS 建立
    ...
    # 通过 Test adapter 或直接写库 + publish bus 触发 message.created
    e2e_db_session.add(new_conv_msg); e2e_db_session.commit()
    publish_incoming()
    # 断言新会话行出现，且 URL 没变（没整页 reload）
    before_url = page.url
    page.get_by_test_id("conv-row", has_text="NewContact").wait_for(timeout=3000)
    assert page.url == before_url
```

E8（console 干净）示范：

```python
@pytest.mark.e2e
def test_no_console_errors(page, e2e_server):
    errors = []
    page.on("console", lambda m: errors.append(m.text) if m.type == "error" else None)
    page.goto(f"{e2e_server}/admin/login")
    page.get_by_test_id("login-token").fill("e2e-token")
    page.get_by_test_id("login-submit").click()
    page.wait_for_url("**/admin")
    page.wait_for_timeout(1000)  # 给 WS onclose 逻辑留窗口
    assert errors == [], f"console errors: {errors}"
```

## 9. 与现有部署钩子的兼容（关键，别破坏 CI）

现有 `.github/workflows/deploy.yml` pre-merge 是：
```
uv run ruff check src/ tests/ && uv run pytest tests/ -v
```
`AGENTS.md` 也记了同一句。**默认 `pytest tests/` 会扫到 `tests/e2e/` 并尝试跑浏览器测试 → CI 没装浏览器 → 全红。**

**方案：marker 隔离 + 钩子微调**

1. `pyproject.toml` 加：
   ```toml
   [tool.pytest.ini_options]
   markers = [
       "e2e: browser E2E tests (require playwright + installed browsers; skipped by default)",
   ]
   ```
2. 所有 E2E 测试加 `@pytest.mark.e2e`。
3. 部署钩子改为（**需同步改 `.github/workflows/deploy.yml` 和 `AGENTS.md` 的 Pre-merge 行**）：
   ```
   uv run ruff check src/ tests/ && uv run pytest tests/ -v -m "not e2e"
   ```
4. 本地/CI 跑 E2E：
   ```
   uv run pytest tests/e2e -v -m e2e
   ```
   或全跑：`uv run pytest tests/ -v -m ""`（清空 marker 过滤）。

**不推荐**用 `addopts = ["-m", "not e2e"]` 写死默认跳过——会让 `-m e2e` 显式调用时产生 marker 组合歧义。改钩子更直白。

## 10. 失败诊断

pytest-playwright 内置，无需额外代码：

```
uv run pytest tests/e2e -v -m e2e \
  --tracing on \
  --video on \
  --screenshot only-on-failure \
  --output=test-results/
```

失败时 `test-results/<test>/` 下有：
- `trace.zip`（Playwright Trace Viewer：`uv run playwright show-trace trace.zip`）
- `video.webm`
- `test-*.png`（失败瞬态）

E2E `conftest.py` 里再统一加一条：每个测试 teardown 时若有 `page.console_errors`，附到 `pytest.fail` 消息里——避免 console 错误被忽略。

## 13. 不做的事（避免过度设计）

- ❌ 不引 Node/TypeScript Playwright——Python 版够用
- ❌ 不重构现有 `tests/` 扁平目录
- ❌ 不用 `.env.test` 切库——沿用 fixture 注入
- ❌ 不把 Playwright MCP 当测试架构核心
- ❌ 不用 Playwright 重写已有 httpx HTML 断言
- ❌ 不做 v2 范畴的跨渠道/admin UI 全功能 E2E
- ❌ 不预留接口/占位测试（遵循 AGENTS.md「不预留接口」原则）
