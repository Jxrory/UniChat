```
  4. 等待 conv-row 保持 visible（确保页面未崩溃）
  5. 断言: URL 没有变化（无整页 reload）
```

**同会话 message-panel 刷新**: `tests/e2e/test_missing.py::TestWebSocketPush::test_ws_push_refreshes_message_panel_same_conversation`

```
步骤:
  1. 登录 → 直访 /admin
  2. 点击现有的 conv-row 打开对应 Conversation 的消息面板
  3. 等待 1s 确保 WebSocket 连接已建立
  4. 通过 page.request.post 发测试 webhook 到 /webhooks/test/tg
```

**不同会话不打扰**: `tests/e2e/test_missing.py::TestWebSocketPush::test_ws_push_does_not_refresh_message_panel_different_conversation`
```
前置: seeded_conversation
步骤:
  1. 登录 → 直访 /admin
  2. 点击现有的 conv-row 打开对应 Conversation 的消息面板
  3. 等待 1s 确保 WebSocket 连接已建立
```

## 四、基础设施说明

### 4.1 测试隔离

```
每个 session（一次 pytest 调用）：
  └─ e2e_server fixture (session-scoped)
       ├─ 创建临时 SQLite 文件
       ├─ create_app(test_config) → uvicorn.run() 在守护线程
       ├─ 等待 /health 就绪
       └─ 返回 base_url

每个测试函数：
  └─ _clean_db fixture (autouse, function-scoped)
       ├─ 清理所有 4 张表（DELETE FROM）
       ├─ 测试执行
       └─ 再次清理所有表
  └─ seeded_conversation fixture (opt-in)
       ├─ 插入 1 条 contact + 1 条 contact_inbox + 1 条 conversation + 1 条 message
       └─ 返回 { conv_id, contact_id }
```

### 4.2 数据库种子数据

Contacts:
- id: `e2e-contact-1`, source_id: `e2e-user-1`, name: "E2E User"

ContactInboxes:
- id: `e2e-ci-1`, inbox_id: `tg`, source_id: `e2e-user-1`

Conversations:
- id: `e2e-conv-1`, inbox_id: `tg`, status: `active`

Messages:
- 1 条 incoming messages: "Hello from E2E test!" (sender: contact)

所有时间戳固定为 `2026-07-07T10:00:00+00:00`，避免时区相关断言问题。

### 4.3 服务器配置

- 端口: 随机空闲（`_find_free_port()`），无冲突
- admin_token: `e2e-token`（与 seed 数据一致）
- channel_type: `test`（不依赖真实 Telegram API）
- DB: 临时文件 SQLite（非内存，TCP 可达）

## 五、data-testid 映射表

所有模板交互元素均标注 `data-testid`，Playwright 通过 `get_by_test_id()` 定位。

| data-testid | 元素 | 所在模板 | 用途 |

## 六、运行方式

### 6.1 本地运行（开发者机器）

```bash
# 首次需安装浏览器
uv run playwright install chromium

# 跑所有 E2E 测试
uv run pytest tests/e2e -v -m e2e

# 跑带失败诊断的完整 E2E
bash script/run_e2e.sh

# 指定单个测试
uv run pytest tests/e2e/test_smoke.py -v -m e2e -k "test_login"

# 指定新增缺失测试
uv run pytest tests/e2e/test_missing.py -v -m e2e
```

### 6.2 CI 自动化

在 CI 中运行 E2E：
```bash
uv run playwright install chromium --with-deps
uv run pytest tests/e2e -v -m e2e
```

### 6.3 失败诊断

Playwright 内置诊断输出：

```
test-results/
└── <test-name>/
    ├── trace.zip        # uv run playwright show-trace trace.zip
    ├── video.webm       # 录屏回放
    └── test-*.png       # 失败瞬态截图
```

命令行使用 `script/run_e2e.sh` 自动生成以上文件。

## 七、注意事项

1. **E2E 不做 HTML 内容断言**: 具体文案留给 L1 测试，E2E 只断言"元素可见、swap 发生、console 干净"。
2. **数据隔离**: 每个测试函数自动清表，`seeded_conversation` fixture 每次都重新写入种子数据。
3. **端口不冲突**: 使用 OS 分配的随机端口（`port=0`），可并行运行。
4. **不依赖外部服务**: 使用 Test adapter（`channel_type="test"`），不调真实 Telegram API。
5. **CI 默认跳过**: `-m "not e2e"` 确保部署钩子不跑浏览器测试。
